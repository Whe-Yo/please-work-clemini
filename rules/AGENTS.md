# Clemini — 오케스트레이션 규율

Claude(머리)가 Gemini(손)에 `agy` CLI로 작업을 위임할 때의 규칙. **제미나이는 가성비가 좋으니 아끼지 말고 쓴다 — 단 호출 오버헤드 때문에 "자주, 크게 묶어서"가 원칙.**

## 라우팅 — 뭘 누구에게

| 작업 | 담당 | 비고 |
|---|---|---|
| 조사·자료 수집·코드베이스 탐색 | **Gemini** | 큰 컨텍스트·싼 토큰의 강점 |
| 1차 논리 검증·안티테제(반론) | **Gemini** | 독립 검토자로 먼저 돌림 |
| 대량 실행·기계적 수정·요약 | **Gemini** | 부피는 싼 손이 |
| 오케스트레이션·명세 작성 | **Claude** | 닫힌 명세로 위임 준비 |
| **최종 판단·채택/기각·치명적 검증** | **Claude** | 제미나이 결과를 통합·재검증 |

핵심: 제미나이를 **1차 일꾼이자 1차 검토자**로 폭넓게 쓰고, Claude는 **명세 + 최종 판단**에 토큰을 집중(효율화 — 중복·장황 배제).

## Always do
- 위임은 `bin/delegate.sh`(= `agy -p`)로. 조사·검토는 `--approval-mode plan`, 좁은 편집은 `auto_edit`.
- **호출은 묶어서 크게.** agy는 호출당 ~23–25k 토큰 오버헤드 → 작은 위임 여러 번 ❌, 한 번에 충분한 맥락·종료조건 담아 적게 ⭕.
- 제미나이 1차 결과(조사·반론)는 **Claude가 통합·최종 판단**한다. 채택은 Claude.
- 명세엔 종료조건을 박는다: "X 되면 멈춰". 체크리스트로.
- Claude 토큰은 효율화: 이미 확립된 사실 재도출·장황한 서베이 금지, 추천 1개+근거로.

## Ask first
- `auto_edit`로 파일 수정 위임 전 — 범위(디렉토리·파일) 확인.
- 위임 범위가 아키텍처에 닿을 때.

## Never do
- `agy --approval-mode yolo` — 승인 게이트 제거 = 폭주(omega 루프) 부활. Gemini는 열린 작업에서 무조건 미끄러진다(실증).
- "개선/최적화/더 좋게" 같은 **열린 동사로 위임** — 폭주 방아쇠.
- 제미나이의 **1차 검증을 최종으로 채택** — 독립 1차 반론은 좋지만 최종 적부는 Claude가. 치명적·보안 검증은 Claude·결정론적 도구로.
- 비밀(.env·토큰)을 명세나 결과 로그에 포함.

## 적용 — please-work-claude 위에서
Claude는 [please-work-claude](https://github.com/Whe-Yo/please-work-claude) 규율(boost·RPW·antithesis) 하에 동작하고, clemini는 그 위의 **위임 층**이다. 안티테제도 "독립 인스턴스" 한 자리를 Gemini가 싸게 채울 수 있다(Claude가 최종 통합).
