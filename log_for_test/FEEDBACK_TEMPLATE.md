# Feedback 작성 템플릿 (clemini)

clemini를 실사용하다 발견한 문제·개선점을 이 폴더(`log_for_test/`)에 md로 남긴다.
파일명: `{YYMMDD_HHMM}_{에이전트}_feedback.md` (검토 로그와 구분되게 `_feedback`).

```
# Feedback — {YYMMDD_HHMM} / {에이전트}

## 유형
bug | enhancement | friction

## 요약
(한 줄)

## 무슨 일이
- 하려던 것:
- 일어난 것:
- 기대한 것:

## 환경
- 에이전트 / OS / 도구 (agy 버전 등):

## 재현
1.
2.

## 관련
(delegate.sh / 라우팅 / 규칙 등)
```

> 비밀(.env·토큰)·프로젝트 작업 내용은 포함하지 않는다. clemini에 관한 피드백만.
