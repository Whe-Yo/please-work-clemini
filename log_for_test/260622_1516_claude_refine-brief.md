# clemini 실증 — --refine/--brief dogfood (260622_1516)

**세션**: bleeding-edge 연구 도메인 1건을 `--refine 2 --brief`로 위임, Claude 라이브 검증. v0.3.0 신기능 실사용.

## 동작 (긍정)
- `--refine 2 --brief`: 2패스 자기개선 + 압축출력 정상. **~1:40**(2 agy 콜), 출력은 **결론 먼저 + 구조적 불릿**으로 Claude가 적게 읽음. 품질·완결성 좋음.
- 다중패스가 구조·누락 보완에 기여(1패스 대비 정리도↑). 환각루프 없음("올바른 부분 보존" 지시 효과).

## 핵심 발견 (패턴 D 강화)
- **`--refine`는 품질·완결성은 올리나 '최신성'은 못 올린다.** Gemini는 정확하지만 **훈련 cutoff 탓에 ~1세대 뒤처진** 그림을 냈고(오래된 대표 방법 위주, "향후 방향"이라 한 게 이미 도래), **Claude 라이브 검증이 현행 SOTA를 보강**해야 완성됐다.
- 즉 refine 횟수를 늘려도 스테일은 안 고쳐진다 — 같은 cutoff 안에서 다듬을 뿐.

## 개선 시사 (휴리스틱 정교화)
- **`--refine` = 구조·완결·정합성** / **Claude 라이브 = 최신성·시점민감 사실**. 둘은 다른 축 → 병행하되 역할 분리.
- 빠르게 바뀌는 도메인(연구 SOTA·가격·상장·일정)은 **refine보다 Claude 검증이 결정적**. rules에 "최신성은 refine으로 못 메운다 — Claude 라이브로" 명시 권장.
